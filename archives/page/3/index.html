剑指offer-最大连续子序列和

2018-12-14
  • 算法

题目描述

给一个数组，返回它的最大连续子序列的和，你会不会被他忽悠住？(子向量的长度至少是1)

解题思路

动态规划问题解题思路 O（n）

`public class Solution {
public int FindGreatestSumOfSubArray(int[] array) {
    int result = array[0];
    int sum = array[0];
    for(int i = 1; i < array.length; i++){
        if(array[i] + sum > result){
            result = sum + array[i];
        }
        if(sum + array[i] < 0){
            sum = 0;
            continue;
        }
        sum = sum+array[i];
    }
    return result;
}

}`

Java基础-逻辑运算符与位运算符

2018-12-13
  • 语言基础

逻辑运算符

  • & 与
  • && 与 左侧为false时 右侧不执行（短路）
  • | 或
  • || 或 左侧为true 右侧不执行（短路）
  • ^ 异或
  • ! 非

位运算符

  • & 与
  • | 或
  • ^ 异或
    • 特点：一个数据对另一个数据位异或两次，该数本身不变。
    • 例：不定义第三方变量，实现两个整数变量交换。
    • 解法1：x = x + y; y = x - y; x = x - y;
    • 解法2：x = x ^ y; y = x ^ y; x = x ^ y;
  • ~ 按位取反
  • << 左移 左侧最高位丢弃，右边补齐0（左移几次就是乘以2的几次幂）

  • 右移 最高位是0，左边补0；最高位是1，左边补齐1（右移几次就是除2的几次幂）

  • 无符号右移 无论最高位是0还是1，左边补齐0

  • 三目运算符 （关系表达式） ？ （关系表达式1） : (关系表达式2)；

linux c/c++-头文件及动态链接库加载问题

2018-12-12
  • 语言基础
  • 头文件加载
    • -I /usr/local/include 表示将/usr/local/include作为第一个寻找头文件的目录，当用到
  • 动态链接库
    • -L /usr/local/lib 表示将 /usr/local/lib 目录作为第一个寻找库文件的目录，寻找的顺序是：/usr/local/lib–>/lib–>/usr/lib
    • -lmemcached表示在上面的lib的路径中寻找libmemcached.so动态库文件（如果gcc编译选项中加入了“-static”表示寻找libmemcached.a静态库文件），程序链接的库名是memcached
    • 链接器ld的选项有 -L，-rpath 和 -rpath-link，看了下 man ld，大致是这个意思：
      • -L: “链接”的时候，去找的目录，也就是所有的 -lFOO 选项里的库，都会先从 -L 指定的目录去找，然后是默认的地方。编译时的-L选项并不影响环境变量LDLIBRARYPATH，-L只是指定了程序编译连接时库的路径，并不影响程序执行时库的路径，系统还是会到默认路径下查找该程序所需要的库，如果找不到，还是会报错，类似cannot open shared object file。
      • -rpath-link：这个也是用于“链接”的时候的，例如你显示指定的需要 FOO.so，但是 FOO.so 本身是需要 BAR.so 的，后者你并没有指定，而是 FOO.so 引用到它，这个时候，会先从 -rpath-link 给的路径里找。
      • -rpath: “运行”的时候，去找的目录。运行的时候，要找 .so 文件，会从这个选项里指定的地方去找。对于交叉编译，交叉编译链接器需已经配置 –with-sysroot 选项才能起作用。也就是说，-rpath指定的路径会被记录在生成的可执行程序中，用于运行时查找需要加载的动态库。-rpath-link 则只用于链接时查找。
  • 当遇到不同机器相同系统下，链接通过但无法找到加载库时，考虑使用-rpath加链接库路径，援引我的老师的一句话，解释了我为什么遇到相同系统环境，链接库加载情况不同。

    因为虽然都是FreeBSD-11.2, 但是因为CPU不同是两个重新编译的内核
    a服务器 是Freebsd-11.2 for X86 因为它不支持gelara数据为同步，所以购买服务器时购买了AMD的服务器， b服务器是Freebsd-11.2 for AMD

Java基础-命名规则

2018-12-12
  • 语言基础

包

其实就是文件夹 用于解决相同类名问题
包名要全部小写 一般用公司域名倒着写
com.xxxx.www

类或接口

一个单词或多个单词分别举例
如果是一个单词，要求首字母大写
如果是多个单词，要求每个单词首字母大写（驼峰命名）
Adapter
ArrayAdapter

方法和变量

如果是一个单词，每个字母都小写
如果是多个单词，从第二个单词开始首字母大写
test（）
testFunction（）

常量

如果是一个单词，所有字母大写
如说是多个单词，也是所有字母大写，但是用_分开
MAX
MAX_VALUE

Java基础-基本数据类型

2018-12-12
  • 语言基础
  • 基本数据类型（4类8种）
    • 整数型
      • byte 占一个字节 -128 到 127
      • short 占两个字节 -2^15 到 2^15-1
      • int 占四个字节 -2^31 到 2^31-1
      • long 占八个字节 -2^63 到 2^63-1
    • 浮点型 默认为double （浮点型为近似表示法，无法准确等于某个小数，只能是无限逼近，尽量避免浮点型之间的比较，比较时可以引入BigDemical，但会降低效率。
      所以，要比较浮点数是否相等，需要做的事情是：1.排除NaN和无穷 2.在精度范围内进行比较）
      • float 占四个字节 1位符号位 8位指数位 23位尾数或有效数字（基于IEEE754）
      • double 占八个字节 1位符号位 11位指数位 52位尾数或有效数字（基于IEEE754）
    • 字符型
      • char 占两个字节 0~65535
    • 布尔型
      • boolean
        • boolean 理论上是占八分之一个字节，一个开关可以决定TRUE或False，但是Java中boolean 类型没有明确指定大小
